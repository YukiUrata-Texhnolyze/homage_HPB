# 美容系予約サイト構築プロジェクト

## 🎯 目的
ホットペッパービューティーのような美容系予約サイトを構築し、店舗とユーザーをつなぐプラットフォームを提供する。

---

## 🧭 プロジェクトの流れ

### 1. サービス設計
- ターゲット業種の選定（美容室 / ネイル / エステ など）
- ユーザー層の定義（店舗側・利用者側）
- 収益モデルの検討（掲載料 / 予約手数料 / 広告収入 / サブスク）

### 2. 競合分析
- ホットペッパービューティー
- minimo
- 楽天ビューティー
- EPARK
- 各サービスの機能・UI・収益構造を比較

### 3. 機能要件定義（MVP）
#### ユーザー向け
- 店舗検索・絞り込み
- メニュー表示
- 空き状況確認
- 予約フォーム
- 会員登録・ログイン
- 口コミ・評価投稿

#### 店舗向け
- 店舗情報登録・編集
- メニュー管理
- 予約管理（カレンダー連携）
- 顧客管理
- 管理画面（ダッシュボード）

#### 管理者向け
- 掲載店舗の承認
- ユーザー・店舗管理
- 売上・アクセス分析

### 4. 技術選定
- フロントエンド：Next.js（React）
- バックエンド：Node.js + Express / Firebase
- データベース：PostgreSQL / Firestore
- 認証：Firebase Auth / Auth0
- 決済：Stripe API
- インフラ：Vercel / AWS / GCP
- 開発支援：GitHub Copilot

### 5. UI/UX設計
- Figmaでプロトタイプ作成
- モバイルファースト設計
- 店舗・ユーザー両方の使いやすさを重視

### 6. 開発フェーズ
- GitHubでリポジトリ管理
- Copilotを活用して機能ごとに開発
- テスト：Jest / Playwright
- CI/CD：GitHub Actions
